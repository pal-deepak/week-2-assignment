1. <b> or <strong>:
    • Use to highlight important or emphasized text.
    • Avoid using it for purely visual purposes; use CSS for styling instead.
    • Do not use for headings; use appropriate heading tags like <h1> to <h6>.
  2. <i> or <em>:
    • Use to indicate text in an alternate voice, such as a book title or foreign word.
    • Avoid using it solely for styling purposes like making text italic; use CSS for that.
  3. <u>:
    • Use to underline text to convey importance or to represent a hyperlink if consistent with design patterns.
    • Avoid using it solely for decorative purposes; use CSS for styling.
  4. <s> or <strike>:
    • Use to indicate deleted or no longer accurate information.
    • Avoid using it for styling purposes like striking through text; use CSS instead.
  5. <sub>:
    • Use to display text as subscript, such as mathematical or chemical formulas.
    • Avoid using it solely for styling purposes; use CSS for visual adjustments.
  6. <sup>:
    • Use to display text as superscript, such as for footnotes or exponentiation.
    • Avoid using it solely for styling purposes; use CSS for visual adjustments.
  7. <mark>:
    • Use to highlight or indicate a specific portion of the text with a background color.
    • Avoid using it excessively or for non-essential elements to maintain readability.
  8. <small>:
    • Use to render text in a smaller font size, typically for disclaimers or fine print.
    • Avoid using it for general styling purposes; use CSS for font size adjustments.
  9. <big>:
    • Use to render text in a larger font size, although this tag is deprecated in HTML5.
    • Avoid using it for general styling purposes; use CSS for font size adjustments.
  10. <code>:
    • Use to represent a fragment of computer code, typically within a paragraph.
    • Use appropriate preformatted elements like <pre> or <code> for larger code blocks.
    • Escape HTML entities within the code to prevent rendering issues.
